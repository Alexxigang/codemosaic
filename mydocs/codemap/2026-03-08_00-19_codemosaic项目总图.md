# CodeMap: codemosaic 项目总图

## 项目现状

- 仓库初始状态只有 `README.md`
- 本轮目标是在单仓内搭建一个可运行的 CLI 原型

## 目标结构

- `codemosaic/cli.py`：CLI 入口，负责命令分发
- `codemosaic/policy.py`：策略模型与 YAML/JSON 加载
- `codemosaic/mapping.py`：占位符映射表与落盘
- `codemosaic/maskers/python_masker.py`：Python 词法级脱敏
- `codemosaic/maskers/text_masker.py`：其他语言轻量兜底脱敏
- `codemosaic/workspace.py`：目录扫描、文件处理、报告输出
- `codemosaic/patching.py`：patch 回填与应用
- `tests/`：基础行为验证
- `docs/`：设计、威胁模型与架构说明

## 核心数据流

1. 用户执行 `mask`
2. CLI 加载策略
3. Workspace Runner 遍历源目录
4. 按文件选择 Python masker 或文本 masker
5. Mapping Vault 为敏感 token 分配占位符
6. 输出 masked workspace 与 mapping/report
7. 用户把 masked code 发给外部 AI
8. 外部 AI 返回 patch
9. 用户执行 `unmask-patch`
10. Patch Translator 用 reverse mapping 恢复原始 token

## 关键约束

- 不引入第三方依赖，方便离线和快速启动
- 先保证可运行、可验证，再演进到 AST 级精准实现
- 默认安全优先，不处理的文件不自动复制
