The Problem

Teams are stuck between AI leverage and code exposure risk.

Most teams want the productivity benefit of external AI coding tools, but do not want to paste internal APIs, proprietary business logic, or secret-bearing source files into third-party systems.

The real question is not just "did we hide the secrets?". It is also "after masking, is this code still safe enough to send?".

What a usable solution must do

  • Preserve enough structure for AI to remain helpful
  • Hide sensitive names, strings, and comments
  • Support reversible patch workflows
  • Keep the mapping vault local
  • Decide whether masked code may be exported

The Solution

CodeMosaic adds a policy gate to the masking workflow.

1

Prepare

Scan and mask a workspace before any code leaves the machine.

2

Evaluate

Run semantic leakage analysis to estimate how much business meaning still remains.

3

Govern

Use Leakage Budget Gate to allow or block exports based on policy thresholds.

4

Recover

Translate masked patches back onto the original repository when external AI returns a fix.

Why It Stands Out

This is not just secret redaction.

Reversible workflow

CodeMosaic supports a full path from masking to patch translation instead of stopping at one-way export.

Semantic leakage analysis

It estimates how much business meaning still survives after masking and ranks the riskiest files.

Managed-key policy model

Teams can use policy-linked key sources, record key IDs, and rotate mapping protection more like a real product.

Demo Flow

Show both success and governance.

  1. Run `scan` on a repo with internal URLs, identifiers, and comments.
  2. Run `mask` and inspect the local mapping vault.
  3. Generate a leakage report and show the highest-risk file.
  4. Build a normal bundle and then show a blocked safe export path.
  5. Translate a masked patch back into the original repo.

Starter commands

python -m codemosaic generate-key --output ./.codemosaic/keys/team-dev.key
python -m codemosaic mask ./repo --policy policy.sample.yaml
python -m codemosaic leakage-report ./repo.masked --policy policy.sample.yaml --output leakage-report.json
python -m codemosaic bundle ./repo.masked --policy policy.sample.yaml --output ai-bundle.md --leakage-report leakage-report.json --fail-on-threshold

Presets

Start with a policy profile, then tighten.

  • Strict: internal platforms, core algorithms, finance, security-sensitive services
  • Balanced: most application repos and first external-AI pilots
  • Public SDK: semi-public integration code and sample repos

Suggested starting presets

python -m codemosaic list-policy-presets
python -m codemosaic init-policy --preset balanced-ai-gateway --output policy.codemosaic.yaml

Built-in presets: strict-ai-gateway.yaml, balanced-ai-gateway.yaml, and public-sdk-ai-gateway.yaml.

Use docs/ci-governance.md and examples/github-actions/leakage-gate.yml to turn these into team workflows.

Ideal Users

Built for teams adopting external AI carefully.

  • Teams with private repositories and external AI coding workflows
  • Security-conscious engineering orgs that want a local-first starting point
  • Companies not ready for a full enterprise DLP or self-hosted model stack

Open Source Direction

What exists now, and what comes next.

  • Now: local masking, encrypted mappings, leakage analysis, export gating, VS Code prototype
  • Next: stronger AST masking, richer policy review UX, CI governance recipes, more polished IDE experience

CodeMosaic helps teams use external AI coding tools without giving up control over what their code reveals.
