The Problem

Teams are stuck between AI leverage and code exposure risk.

Most teams want the productivity benefit of external AI coding tools, but do not want to paste internal APIs, proprietary business logic, or secret-bearing source files into third-party systems.

The real question is not just “did we hide the secrets?”. It is also “after masking, is this code still safe enough to send?”

What a usable solution must do

  • Preserve enough structure for AI to remain helpful
  • Hide sensitive names, strings, and comments
  • Support reversible patch workflows
  • Keep the mapping vault local
  • Decide whether masked code may be exported

The Solution

CodeMosaic adds a policy gate to the masking workflow.

1

Prepare

Scan and mask a workspace before any code leaves the machine.

2

Evaluate

Run semantic leakage analysis to estimate how much business meaning still remains.

3

Govern

Use Leakage Budget Gate to allow or block exports based on policy thresholds.

4

Close the loop

Translate masked AI patches back to original code and apply them safely.

Why It Stands Out

Not just redaction. Export governance.

Reversible closed loop

mask -> AI -> patch -> unmask/apply keeps the workflow practical, not theoretical.

Semantic leakage analysis

Measures what business meaning still leaks after masking, not only what looked secret before masking.

Leakage Budget Gate

Blocks unsafe exports locally before masked code leaves the workstation boundary.

Policy-driven segmentation

Apply different rules to different paths and split masking runs into governed segments.

VS Code safe export UX

Developers can trigger safe export directly in the editor and see gate outcomes in the Runs view.

Get Started

Pick the path that fits your audience.

  • Developers: clone the repo and run the CLI locally
  • Evaluators: install the VSIX from Releases and run the demo workflow
  • Security reviewers: open `docs/leakage-gate.md`, `docs/landing-page.md`, and `RELEASE_CHECKLIST.md`

Install path

python scripts/package_vscode_extension.py --overwrite
# or download a VSIX from GitHub Releases

python scripts/generate_release_assets.py --encrypt-mapping --clean

Use the generated VSIX for editor demos and the release assets for product-facing walkthroughs.

3-Minute Demo

Show both outcomes: normal bundle and blocked safe export.

  1. Run Scan Workspace
  2. Run Mask Workspace
  3. Open Analyze Semantic Leakage
  4. Generate a normal AI bundle
  5. Run Safe Export Bundle
  6. Show the gate decision and top-risk file
  7. Open the Runs view and point at Leakage gate: BLOCKED

Safe export command

python -m codemosaic bundle ./your-repo.masked \
  --policy policy.sample.yaml \
  --output ai-bundle.md \
  --leakage-report leakage-report.json \
  --fail-on-threshold

If the leakage budget is exceeded, CodeMosaic exits with code 3 and does not generate the bundle.

Policy Presets

Start with a preset instead of designing policy from scratch.

  • Strict: internal platforms, core algorithms, finance, security-sensitive services
  • Balanced: most application repos and first external-AI pilots
  • Public SDK: semi-public integration code and sample repos

Suggested starting presets

presets/strict-ai-gateway.yaml
presets/balanced-ai-gateway.yaml
presets/public-sdk-ai-gateway.yaml

Use `docs/ci-governance.md` and `examples/github-actions/leakage-gate.yml` to turn these into team workflows.
