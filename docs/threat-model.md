# Threat Model

## 保护目标

- 核心算法命名
- 内部域名、接口地址、租户信息
- 密钥、Token、账号、邮箱、手机号
- 含业务语义的注释和常量

## 信任边界

- `trusted`: 本地开发机、原始仓库、mapping vault
- `semi-trusted`: masked workspace
- `untrusted`: 外部 AI 平台、第三方模型服务

## 攻击面

- 直接把明文源码发送给外部模型
- prompt 中额外暴露业务背景
- patch 回填过程中把 mapping 文件一并泄露
- 过于结构保真的 masked code 被模型语义反推

## 当前缓解手段

- mapping 只本地落盘
- 默认不复制未处理文件
- 支持按目录和文件类型控制发送范围
- 注释与字符串默认脱敏

## 剩余风险

- 占位符周围的控制流仍可能暴露业务逻辑
- 函数数量、调用关系和目录结构仍然可能泄露信息
- AI 可能根据上下文猜测原语义
- 当前版本未对 mapping 进行加密

## 使用建议

- 只发送本次任务最小必要文件
- 不要共享 `.codemosaic/` 目录
- 对高敏项目继续配合企业 DLP 与访问审计
- 对涉密算法优先考虑内网模型或自托管模型
